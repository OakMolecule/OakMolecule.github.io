C/C++ const修饰符
==============

写一篇简单点的，最近老是使用const，但是还经常想不起来咋用，老得查，写一篇记录以下，方便下次找。

作用
--

`const` 说白了就是希望某些值不被改变，对于一般的变量来说很简单 `const int i = 1` 即可，但是对于指针来说就没那么简单了。

### 指针

``` C
const int *i = 0;      // 值不能改，指向可以随便改
int *const j = 0;      // 值可以随便改，指向不能改
const int *const k = 0;
```

前两个很容易分不清楚，首先第一个是指向常量的指针，说人话就是 `i` 指向的值不能通过 `*i` 改变，但是可以通过别的方法改变，并且 `i` 这个指针可以随便指向其他变量。 第二个是常量指针，即指针不能随便指别人，但是指向的值可以随便改。 第三给就是都不能改了。

`k` 这一种（`const int *const k`）可以理解为两层都加了 `const`：
- `k` 不能指向别的地址（指针本身是 const）
- `*k` 的值也不能通过这个表达式被修改（所指内容是 const）

### 引用（const）

`const` 和引用经常一起出现。常见写法是用 `const` 引用作为函数参数，表示“函数内部不应该修改传入的对象”：

``` C
void f(const int& r) {
    // r 不能通过 r 被修改
}
```

### 成员函数的 const

类的成员函数也可以加 `const`，表示该成员函数不会修改对象的成员变量（编译器层面会限制修改）：

``` C
struct A {
    int x;

    void foo() const {
        // x = 1; // 这一类修改通常会直接编译失败
    }
};
```

### 常见误区

- `const` 不是“运行时不可变”，而是“在编译期限制你通过该表达式修改数据”。
- 对指针而言，`const` 贴在哪个符号上，含义就会变了（指针本身 `const` vs 指向内容 `const`）。
